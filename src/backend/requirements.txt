# Hashing de contraseñas
# [SEC] bcrypt se fija en 4.0.1 deliberadamente: passlib 1.7.4 (última
# versión estable al momento de escribir esto) tiene un bug conocido de
# detección de versión con bcrypt>=4.1, que dispara un warning interno
# "error reading bcrypt version" al hashear. No es una falla de seguridad
# (el hash se genera y verifica correctamente), pero es ruido evitable.
# Verificado en este proyecto: bcrypt==4.0.1 + passlib==1.7.4 = sin warnings.
passlib[bcrypt]==1.7.4
bcrypt==4.0.1
